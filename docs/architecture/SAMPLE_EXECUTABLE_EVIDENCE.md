# Sample Executable Evidence

> Last updated: 2026-06-04
> Status: Report-only evidence inventory

`SagaSampleExecutableEvidence` is a bounded build-time evidence target. It is
included in the default CMake build when
`SAGA_ENABLE_SAMPLE_EXECUTABLE_EVIDENCE=ON` and writes a machine-readable sample
executable evidence report under the active build tree.

The target depends on real native executables: `SagaRuntime`,
depends on the real `Saga` product executable.

The report is generated by `scripts/verify-sample-executable-evidence` from
real commands. It records `verified: false` and can report
`passed-with-limitations`, `blocked`, or `failed`.

This document is not a product or architecture readiness claim. It records the
scope of one evidence target so current product and architecture docs do not
need to carry local report-path details.

## Evidence Scope

`StarterArena` evidence covers project validation, SagaScript analyze/compile,
the compiled script assembly, and one controlled
`SagaRuntime --starter-arena-smoke --invoke-starter-arena-script` run. The
runtime smoke records `GameRules.AddPickupScore(10, 5)` evidence.

`MultiplayerSandbox` evidence covers project validation, SagaScript analysis,
Visual Blocks/project-blocks projection behavior, and the current
`MultiplayerSandboxHeadless` smoke path. The report treats this as
fixture/projection/headless evidence, not playable multiplayer game evidence.

## Non-Claims

This evidence is not production-ready, not enterprise-ready, not a verified
final release, not full gameplay readiness, not full editor workflow, not full
Visual Blocks UI, not full collaboration, not proof that `MultiplayerSandbox`
is a playable multiplayer game, and not proof that `StarterArena` is a finished
playable game.
