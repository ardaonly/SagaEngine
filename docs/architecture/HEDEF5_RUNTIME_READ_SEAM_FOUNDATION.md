# Hedef 5 Runtime Read Seam Foundation

Hedef 5 Phase 156-160 is a report-only planning foundation for a future
Runtime read seam. It consumes Hedef 4 closure evidence from
`Build/SourceTruth` and may emit preview readiness evidence under
`Build/Preview`.

This foundation is not Product Beta, not a Release Candidate, not Production
Readiness, not a Playable Editor, not full Runtime gameplay, and not full
Server gameplay. It does not implement Client Preview, ClientHost, Runtime
gameplay loading, Server gameplay, asset import, asset cook, Editor UI, or Qt
UI.

## Phase 156 Scope Boundary

Phase 156 freezes the claim boundary around Runtime read seam contracts only.
The following Hedef 4 residual debt remains active:

- Client Preview remains deferred.
- ClientHost remains planning-only and untouched.
- Asset import and asset cook remain deferred.
- Editor UI and Qt UI remain deferred.
- Runtime gameplay and Server gameplay remain deferred.
- Raw full CTest, heavy stress, soak, bot swarm, and real transport proof
  remain unclaimed or deferred.

## Phase 157 Runtime Read Contract

Future Runtime-readable inputs are:

- scene and entity source truth declared by the project manifest;
- accepted asset references from source truth gate evidence;
- non-canonical generated projections only as evidence;
- freshness evidence proving whether generated projections are fresh, partial,
  missing, stale, or invalid.

Generated projections are consumable evidence only when freshness is proven or
explicitly partial. They never own Runtime source truth.

Forbidden Runtime source truth owners are generated artifacts, package
summaries, launch summaries, editor diagnostics, ClientHost state, stale
projections, and asset import/cook outputs.

## Phase 158 Adapter Audit

The report-only adapter audit reads required evidence from `Build/SourceTruth`:

- source truth inventory and gate;
- scene/entity validation;
- asset reference gate;
- generated artifact freshness gate;
- Runtime readiness gate;
- package source-truth alignment;
- launch source-truth alignment;
- Hedef 4 closure.

Missing required evidence maps to `MissingEvidence`. Failed or blocked required
evidence maps to `Blocked`. Partial evidence maps to `PartiallyProven`.
Complete required evidence maps to `ReadyForImplementationPlanning`, which only
permits future implementation planning.

The audit records missing adapter seams for scene source truth, entity source
truth, component metadata, accepted asset references, generated projection
evidence, and freshness evidence. It does not touch Runtime code.

## Phase 159 Fixture Contract

Minimum future fixture inputs are one validated scene, one entity, component
metadata, accepted asset references, projection metadata, and expected freshness
metadata.

Invalid fixtures block implementation planning. Stale projections remain
partial evidence. Missing freshness evidence blocks implementation planning.
Fixtures are not loaded in Runtime during Phase 156-160, and projections are not
generated by this work.

## Phase 160 Readiness Report

The optional SagaProjectKit command is:

```sh
sagaproject runtime-readiness-v2 --project <.sagaproj> --evidence-root <Build/SourceTruth> --out <Build/Preview/runtime_readiness_v2_report.json>
```

The report emits one of `ReadyForImplementationPlanning`, `PartiallyProven`,
`Blocked`, or `MissingEvidence`. It preserves these invariants:

- `localOnly=true`
- `networkExposure=None`
- `mutatesSource=false`
- `enforcement=ReportOnly`

`ReadyForImplementationPlanning` does not mean Runtime, ClientHost, or Client
Preview exists. It means the evidence set is complete enough to plan the
future Runtime read seam implementation.
