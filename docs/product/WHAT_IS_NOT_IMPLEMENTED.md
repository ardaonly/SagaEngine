# What Is Not Implemented

This page lists the main gaps to know before treating SagaEngine as a product.

## Runtime And Gameplay

- No complete runtime gameplay loop.
- No complete server gameplay loop.
- No finished Client Preview flow.
- No Runtime read seam implementation; existing source-truth/readiness reports
  are planning and diagnostics evidence only.
- No production networking or scale validation.
- No performance-ready MMO product.

## Editor And Authoring

- No finished editor workflow.
- No complete visual scripting product.
- No arbitrary C# to blocks round trip.
- No complete editor-driven scene, prefab, asset, or gameplay workflow.
- No cloud collaboration product.

## Assets And Samples

- No production asset import/cook pipeline.
- No production renderer or visual output guarantee.
- No source-truth evidence should be read as asset import, asset cook, runtime
  asset loading, or visual sample completion.
- `MultiplayerSandbox` is not a finished game.
- Existing sample material is primarily useful as deterministic fixture data.

## Evidence And Diagnostics

- No production observability system.
- No remote telemetry platform.
- No crash-safety, OS memory profiling, leak detection, benchmark-suite, or
  performance-readiness guarantee.
- Local reports are review evidence, not enforcement or source truth.

## Release And SDK

- No public SDK.
- No distribution ready for production use.
- No installer, updater, marketplace, signing, or support workflow.
- No production cloud, account, permission, enterprise security, compliance, or
  support model.
