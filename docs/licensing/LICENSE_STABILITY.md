# License Stability

> Status: DRAFT — policy guidance, not a bulk legal grant.

1. Historical tags keep the licensing state present in their source snapshot.
2. A path move does not itself change a file's license.
3. A target license in `LICENSE_POLICY.toml` is non-effective until an
   effective migration commit is recorded.
4. Canonical license texts must remain byte-verifiable.
5. Vendor and submodule terms are not replaced by repository defaults.
6. Removal of a domain records lifecycle history; it does not rewrite history.
7. License changes require provenance, authority, metadata, validation,
   documentation, and distribution-surface review.
