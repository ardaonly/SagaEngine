# Internal Documentation

This directory contains historical, recovery, and planning material that should
not be presented as current product documentation.

Internal docs can be useful when tracing why a tool, report, or roadmap exists.
They should not be used for external claims unless current product docs also
support the claim.

## Areas

- [product-history/](product-history/) - older product-track, alpha,
  technical-preview, tutorial, and closure documents preserved for context.
