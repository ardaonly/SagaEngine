# Internal Documentation

This directory contains historical and planning material that should not be
presented as current product documentation.

Internal docs can be useful when tracing why a tool, report, or planning note
exists.
They should not be used for external claims unless current product docs also
support the claim.

## Areas

- [product-history/](product-history/) - older product-track, alpha,
  technical-preview, tutorial, and closure documents preserved for context.
- [architecture-appendices/](architecture-appendices/) - proposed architecture,
  implementation-history, and design-rationale appendices that are not current
  architecture entry points.
