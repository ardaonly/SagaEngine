# MultiplayerSandbox Small-Team Alpha Tutorial

Status: Phase 94 onboarding evidence for the MultiplayerSandbox local team path.

The supported project is `samples/MultiplayerSandbox`. The tutorial path is a
local report workflow, not hosted collaboration or realtime multiplayer editor
state.

## Local Team Flow

1. Validate the project and collect `project_validation_report.json`.
2. Analyze/projection-check the C# authoring surface and collect
   `projection_report.json`.
3. Review supported patch/diff evidence through report files.
4. Run the accepted server/headless launch path and collect
   `launch_preview_report.json`.
5. Summarize diagnostics into `diagnostics_summary.json`.
6. Run package publish-check evidence and collect `publish_report.json`.
7. Capture local/offline collaboration metadata reports for sessions, locks,
   comments, review requests, and Team Room status.
8. Run the Small-Team Alpha acceptance, matrix, and closure commands.

## Boundaries

The tutorial does not require ClientHost, two-client preview, Runtime gameplay,
Server gameplay, scene/entity authoring, asset import/cook, editor UI, or Qt UI.

## Non-Claims

- no product beta
- no release candidate
- no production MMO server
- no complete visual scripting
- no arbitrary C# roundtrip
- no enterprise readiness
