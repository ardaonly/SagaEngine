# Saga Release Vocabulary And Claim Levels

Status: Phase 63 Technical Preview limitations and non-claims freeze.

This document defines how Target 1 / Technical Preview may be described after Block J evidence is
collected.

## Accepted Vocabulary

`SagaEngine 0.1 Technical Preview` means:

- one repository-local developer workflow;
- one canonical sample project, `MultiplayerSandbox`;
- focused local evidence for validation, launch, diagnostics, scripting,
  package proof, view honesty, and collaboration metadata;
- reports that can be inspected without claiming broader product maturity.

`Technical Preview candidate` means the Phase 60-65 evidence set is being
assembled or audited.

`Accepted Technical Preview` means Phase 65 closure passed with focused local
evidence and a recorded closure report.

## Required Non-Claims

- no beta product status;
- no release candidate;
- no production MMO server;
- no complete visual scripting;
- no arbitrary C# roundtrip;
- no enterprise readiness.

## Additional Blocked Claims

These remain outside Target 1 / Technical Preview:

- production network readiness;
- arbitrary C# conversion into editable blocks;
- graph editing as a product surface;
- source-changing patch operations;
- full editor MVP;
- full collaboration;
- enterprise collaboration readiness;
- cloud workspace platform;
- full security model;
- MMO-scale proof;
- installer, signing, updater, marketplace, or store publishing.

## Evidence Rule

Every allowed claim must point to one of the focused reports, tests, or closure
documents named by the Technical Preview closure report. If evidence is missing,
the correct status is `Blocked`, not accepted.
